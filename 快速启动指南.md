## 🔧 内网访问问题解决方案

### 问题：阿里内网同事无法访问

**原因：**
- 公司防火墙限制 5000 端口
- VLAN 隔离导致不同网段
- macOS 防火墙阻止入站连接

**解决方案（按推荐程度）：**

#### 方案 A：Tailscale 虚拟局域网 ⭐⭐⭐⭐⭐

最推荐！完全不受物理网络限制。

```bash
# 您和同事都执行：
brew install tailscale
sudo tailscale up
# 浏览器登录同一账号

# 查看您的 Tailscale IP
tailscale ip

# 同事通过此 IP 访问
http://[Tailscale IP]:5000
```

**优势：**
- ✅ 无需开放防火墙端口
- ✅ 穿透公司网络限制
- ✅ WireGuard 加密安全
- ✅ 免费（20 台设备 +3 用户）
- ✅ 5 分钟配置完成

详细配置见 `README-TAILSCALE.md`

---

#### 方案 B：关闭 macOS 防火墙测试 ⭐⭐⭐

如果同事在同一办公网：

```bash
# 临时关闭防火墙
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off

# 让同事测试访问
# http://30.78.162.128:5000
```

---

#### 方案 C：运行诊断工具 ⭐⭐⭐

```bash
./check-network.sh
```

显示所有可能的访问地址和故障排查建议。

---

#### 方案 D：云平台部署（长期方案）⭐⭐⭐⭐

```bash
./deploy-heroku.sh
```

获得永久公网 URL，适合外网朋友访问。
